---
title: 'Get Assessment'
api: 'GET /api/assessments/{assessment_id}'
---

Retrieves the complete details and content of a specific saved assessment.

## Authentication

This endpoint requires authentication. Include a valid JWT token in the Authorization header:

```
Authorization: Bearer <your_access_token>
```

## Request

### Path Parameters

<ParamField path="assessment_id" type="integer" required>
  The unique identifier of the assessment to retrieve. This ID is returned when saving an assessment or can be obtained from the [Assessment History](/api/assessments/history) endpoint.
</ParamField>

## Response

<ResponseField name="id" type="integer">
  The unique identifier of the assessment.
</ResponseField>

<ResponseField name="chapter_name" type="string">
  The chapter or topic name for this assessment.
</ResponseField>

<ResponseField name="content" type="object">
  The complete assessment content including all questions, answers, and metadata.
  
  <Expandable title="content structure">
    <ResponseField name="mcqs" type="array">
      Array of multiple-choice questions.
      
      <Expandable title="MCQ structure">
        <ResponseField name="question" type="string">
          The question text.
        </ResponseField>
        
        <ResponseField name="options" type="array">
          Array of answer options (typically 4 choices).
        </ResponseField>
        
        <ResponseField name="correct_answer" type="string">
          The correct answer text.
        </ResponseField>
        
        <ResponseField name="bloom_level" type="string">
          The Bloom's taxonomy level (e.g., "remember", "understand", "apply", "analyze", "evaluate", "create").
        </ResponseField>
        
        <ResponseField name="explanation" type="string">
          Explanation for why the correct answer is correct.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Request

```bash
curl -X GET "http://localhost:8000/api/assessments/42" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

## Error Responses

### 401 Unauthorized

Returned when the Authorization header is missing or contains an invalid token.

```json
{
  "detail": "Not authenticated"
}
```

### 404 Not Found

Returned when the assessment ID doesn't exist or doesn't belong to the authenticated user.

```json
{
  "detail": "Assessment not found"
}
```

## Security

- Users can only access their own assessments
- Attempting to access another user's assessment returns a 404 error (not 403) to prevent information disclosure
- The assessment ID and user ID are both verified in the database query

## Use Cases

- Display full assessment content for review
- Allow users to retake assessments
- Export assessment data for external use
- Share assessment content (with appropriate permission handling)
- Print or generate PDF versions of assessments

## Notes

- The `content` field structure matches the `content_json` saved via [Save Assessment](/api/assessments/save)
- All questions include explanations to support learning outcomes
- Questions are tagged with Bloom's taxonomy levels for educational tracking

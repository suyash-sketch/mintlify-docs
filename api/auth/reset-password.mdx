---
title: "Reset Password"
api: "POST /api/reset-password"
description: "Reset a user's password using a reset token"
---

Resets a user's password using a valid reset token obtained from the forgot password endpoint. The reset token must not be expired and must be of type "reset".

## Request

<ParamField body="token" type="string" required>
  The reset token received from the forgot-password endpoint
</ParamField>

<ParamField body="new_password" type="string" required>
  The new password to set for the account
</ParamField>

## Response

<ResponseField name="message" type="string">
  A success message confirming the password has been reset
</ResponseField>

## Example Request

```bash
curl -X POST http://localhost:8000/api/reset-password \
  -H "Content-Type: application/json" \
  -d '{
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "new_password": "newsecurepassword456"
  }'
```

## Example Response

```json
{
  "message": "Password has been reset successfully"
}
```

## Error Responses

<Expandable title="400 - Invalid token">
  ```json
  {
    "detail": "Invalid token"
  }
  ```
</Expandable>

<Expandable title="400 - Expired token">
  ```json
  {
    "detail": "Reset token has expired"
  }
  ```
</Expandable>

<Expandable title="404 - User not found">
  ```json
  {
    "detail": "User not found"
  }
  ```
</Expandable>

## Token Validation

The reset token is validated to ensure:
- It has not expired
- It has not been tampered with
- It is specifically a "reset" type token (not an access token)
- The user associated with the token still exists
